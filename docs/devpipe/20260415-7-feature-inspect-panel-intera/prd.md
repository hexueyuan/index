# 需求文档 (PRD)

## 基本信息

- 开发类型: 新功能
- 功能描述: 展示面板（InspectPanel）与交互类型分离
- GitHub Issue: #7
- 远程分支: main
- 本地分支: feature-inspect-panel-intera
- 创建时间: 2026-04-15

## 需求背景

当前项目中，告示牌（sign）复用了 `DialogBox`（对话系统）来展示内容。但告示牌本质是"阅读/展示"而非"对话"，两者在语义、视觉风格和交互方式上存在差异：
- 对话框位于屏幕底部、半透明黑底、逐字打字机效果，适合 NPC 对话
- 告示牌应位于屏幕中央、有主题风格背景（木板/羊皮纸）、直接显示全文

混用对话框展示告示牌内容，会导致后续扩展（照片、日记、道具等查看类交互）时难以区分和定制样式。需要将"查看/展示"和"对话"两种交互类型在架构上分离。

## 功能描述

玩家在地图中面对告示牌按下交互键（Space/Enter/A/触屏）时：
1. 屏幕中央弹出一个像素风木板/羊皮纸风格的展示面板（InspectPanel）
2. 面板显示告示牌的标题和正文内容，文字直接全部呈现（无打字机效果）
3. 玩家在面板打开期间被锁定（不可移动）
4. 玩家按下任意交互键或点击屏幕即可关闭面板
5. 面板关闭后玩家恢复移动

与现有 DialogBox 的差异：
- 位置：屏幕中央（而非底部）
- 背景：代码绘制的像素风木板/羊皮纸效果（而非半透明黑底）
- 文字显示：直接显示全文（而非逐字打字机效果）
- 无说话者角色标识（无 speaker name + role color）

## 功能边界

### 本次包含
- 新建 `InspectPanel` 组件，支持 sign（告示牌）类型的内容展示
- `InspectPanel` 数据结构预留 type 字段，支持后续扩展 photo/diary/item 等类型
- 改造 `MainScene` 的交互对象数据结构，区分 dialog（对话）和 inspect（查看）两种交互
- `tryInteract` 根据交互类型分流到 `DialogBox` 或 `InspectPanel`
- 现有"村口告示牌"从 DialogBox 迁移到 InspectPanel
- 代码绘制像素风木板/羊皮纸风格面板背景（无需外部图片素材）

### 本次不包含
- 不实现 photo（照片）、diary（日记）、item（道具）类型的展示逻辑
- 不在地图上新增更多可交互道具（只迁移现有的"村口告示牌"）
- 不修改现有 DialogBox 组件
- 不修改 Tiled 地图文件中已有对象的结构（现有 sign type 和 text 属性保持不变）

## 验收标准

1. 面对"村口告示牌"按交互键后，弹出的是居中的木板/羊皮纸风格面板，而非底部对话框
2. 面板直接显示告示牌全文（无打字机效果），包含标题和正文
3. 按交互键或点击屏幕可关闭面板，玩家恢复移动
4. 面板打开期间玩家被锁定，无法移动
5. 原有的 DialogBox 功能不受影响（为后续 NPC 对话保留）
6. `InspectPanel` 数据结构包含 type 字段，预留扩展能力
7. 手机端（触屏 A 按钮 + 屏幕点击）交互正常工作
