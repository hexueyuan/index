## 需求背景

项目是一个像素风格 RPG 探索游戏，目前已有地图加载、角色移动、碰撞系统，但没有任何 UI 系统。对话系统是所有文字交互（NPC、告示牌、日记、道具等）的基础载体，是第一个需要实现的 UI 组件。后续所有叙事内容都依赖这个对话框组件。

## 功能描述

### 对话框显示

- 屏幕底部显示半透明黑色背景的对话框
- 对话框内显示说话者名字，通过名字颜色区分角色类型：
  - 玩家角色：名字显示为蓝色
  - NPC 角色：名字显示为黄色
- 对话正文统一为白色文字，以打字机效果逐字显示
- 使用像素字体，与游戏整体像素风格一致

### 翻页与关闭

- **PC 端**：按空格键或回车键翻到下一页，最后一页按键后关闭对话框
- **移动端**：点击对话框本身可翻页；同时在虚拟方向键区域增加一个交互按钮（A 键），也可触发翻页
- 打字机动画进行中按键可跳过动画，立即显示当前页完整文字；再次按键才翻页

### 玩家控制锁定

- 对话框显示期间，玩家角色不能移动
- 对话框关闭后，恢复玩家控制

### 对话数据

- 一段对话由多页文字组成，每页可指定说话者名字和角色类型（玩家/NPC）
- 在 MainScene 中集成测试：按空格键弹出一段包含玩家和 NPC 交替对话的固定对话，验证角色区分效果

## 功能边界

### 本次包含

- 对话框 UI 组件（显示、打字机效果、翻页、关闭）
- 说话者名字显示，按角色类型用不同颜色区分（玩家蓝色、NPC 黄色）
- PC 端键盘操作（空格/回车）
- 移动端点击对话框翻页
- 移动端虚拟交互按钮（A 键）
- 打字机效果跳过功能
- 对话期间玩家移动锁定
- MainScene 中的集成测试（固定对话触发）

### 本次不包含

- NPC 交互触发（后续 NPC 系统实现）
- 对话分支/选项系统
- 对话框样式自定义（颜色、大小可配置）
- 对话中的表情/头像显示
- 打字速度可调节
- 音效（打字音效、翻页音效）

## 验收标准

1. 按空格键能弹出对话框，对话框显示在屏幕底部，半透明黑色背景
2. 对话框显示说话者名字，玩家角色名字为蓝色，NPC 角色名字为黄色
3. 文字以打字机效果逐字出现，使用像素字体
4. 打字过程中按空格/回车可跳过动画，立即显示完整文字
5. 已显示完整文字时，按空格/回车翻到下一页
6. 最后一页按键后对话框关闭
7. 对话期间玩家无法移动，关闭后恢复控制
8. 移动端点击对话框可翻页
9. 移动端虚拟方向键区域有交互按钮（A 键），可触发翻页
10. MainScene 中有包含玩家和 NPC 交替对话的测试对话，能看出角色区分效果
